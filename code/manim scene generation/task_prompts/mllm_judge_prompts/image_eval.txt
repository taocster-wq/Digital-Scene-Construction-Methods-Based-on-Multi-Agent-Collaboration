## Dimension 1) Visual Relevance (1–5)

Judge whether visible content in THIS frame is relevant to {description}.

### Evidence Requirements (MANDATORY)

* Your evaluation must cite concrete visible evidence (e.g., “a right triangle with labeled sides a,b,c”, “formula e^{iθ}=cosθ+isinθ is visible”, “no math text/diagram present”).

### Relevance Checklist (STRICT)

Count as present ONLY if clearly readable/recognizable:

* Theorem-related math text (formula/statement/definitions/proof step)
* Theorem-related diagram/graph/axes/geometry figure with labels
* Symbols/notations that match the described concept
* Contextual title/header that matches {description}

### Distraction Penalty (MANDATORY)

Note if the frame is dominated by:

* UI overlays, watermarks, unrelated imagery, decorative elements, empty/transition background

### Score Caps for Visual Relevance (MANDATORY)

* If {image} missing/unreadable → **score = 1**
* If no theorem-related elements are visible → **score ≤ 2**
* If {description} is vague AND the frame lacks explicit theorem cues (no matching labels/formulas/diagrams) → **score ≤ 2**
* If only weak/ambiguous alignment (generic math symbols with no clear connection) → **score ≤ 3**
* If relevant content exists but is incomplete/partial in a way that reduces usefulness → **score ≤ 4**

### Scoring Anchors

* 5: Strongly aligned; key theorem-specific content is clearly visible and helpful in this moment.
* 4: Mostly aligned; minor unrelated elements or slightly incomplete content.
* 3: Some alignment but ambiguous/partial; weakly informative.
* 2: Mostly irrelevant or dominated by distractions; minimal theorem-related content.
* 1: No usable image OR nothing relevant visible.

## Dimension 2) Element Layout (Placement & Size / Overlap / Clarity) (1–5)

Judge readability and composition as a single snapshot ONLY.

### Layout Checklist (STRICT)

A) Placement & Size

* Key content inside safe viewing area, not hugging edges
* Font/diagram size readable at normal viewing
* Clear hierarchy (title/main formula/diagram vs secondary notes)

B) Overlap / Occlusion / Cropping

* No important text/diagram cut off
* No unintended overlaps (labels over lines, text on top of symbols)
* No collisions among labels/symbols

C) Clarity / Readability

* Text/math legible (stroke thickness, spacing)
* Not too cluttered; spacing supports quick parsing
* If blurred/in-transition artifacts exist, penalize

### Score Caps for Layout (MANDATORY)

* If {image} missing/unreadable → **score = 1**
* If any key text/formula/diagram is **cropped** or **unreadable** → **score ≤ 2**
* If frequent overlaps/label collisions/clutter materially reduce readability → **score ≤ 3**
* If minor overlap/crowding exists but understanding remains → **score ≤ 4**

### Scoring Anchors

* 5: Excellent placement, no overlap/cropping, highly readable.
* 4: Good layout; small issues that don’t block understanding.
* 3: Acceptable; minor crowding/overlap reduces comfort.
* 2: Significant layout problems impair understanding.
* 1: Unreadable/missing image input.
