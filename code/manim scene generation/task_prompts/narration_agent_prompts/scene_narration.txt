Mandatory continuity & transitions (must follow)
================

- Treat all Scenes as one continuous video: camera, objects, and concepts must carry over smoothly.
- For each Scene, you MUST explicitly write three items:

  1) ARR_CONTINUITY_IN: One sentence that connects the previous scene’s final frame/conclusion/object state to this scene’s opening.
  2) ARR_CONTINUITY_OUT: One sentence that sets up the next scene’s motivation/question/suspense.
  3) ARR_TRANSITION: A transition narration line PLUS visual strategy, and include the matching NAR_CUE describing how the frame transitions and how objects are kept/transformed.

- Each Scene must be broken into multiple Sub-scenes (SUB_1..k), and the language must flow naturally (not like reading separate fragments).
- Avoid repetitive formulaic openings/closings (no “Hi everyone / to sum up / see you next time” per scene).
- The narration must be:
  - conceptually accurate and step-by-step clear
  - grounded analogies/examples (no random storytelling)
  - guided questions that invite thinking
  - highlighting key points and common mistakes
  - not merely describing visuals, but using visuals to make ideas click

- Every Scene, every Sub-scene, and every cross-scene transition MUST include the required TAGs.

================================================
Allowed TAG dictionary (ONLY these)
=====================

- ARR_SCENE_ROLE: The pedagogical role of this scene in the full video (motivation/definition/derivation/example/comparison/summary/reinforcement/practice/error-correction)
- ARR_CONTINUITY_IN: One-sentence continuity handoff from the previous scene’s end state to this scene’s opening
- ARR_CONTINUITY_OUT: One-sentence hook that sets up the next scene’s driving question/motivation
- ARR_TRANSITION: Narrative transition line + visual strategy (concept hook + visual anchor)
- NAR_GOAL: What learners should be able to do after this sub-scene
- NAR_LINE: Spoken narration line(s), MUST include a time range
- NAR_REASON: Why this teaching move / alignment helps understanding
- NAR_CUE: Animation cue aligned to narration (object/action/timing)

================================================
Timeline & alignment format (strict)
================

- Use ONE continuous relative timeline starting at 00:00 and increasing across the full video (do NOT reset per scene).
- Every narration line must include a time range:
  `NAR_LINE: (00:12-00:18) ...`
- When precise sync is needed, insert a separate NAR_CUE line in this fixed format:
  NAR_CUE: time=00:15  target={object_or_vgroup_name}  action={Create/Write/Transform/FadeIn/FadeOut/Shift/Circumscribe/CameraMove/Wait}  align={start|end}  note={why synchronize here}
- Each Sub-scene MUST end with a bridge line that connects forward naturally; it may include cues for keeping/transforming/fading objects.
- If the input lacks durations, you must estimate reasonably but MUST mark them with 【ASSUMPTION】.

================================================
Audio-first timing rule (must follow)
======================

- The narration audio is the master clock. Animation must never finish earlier than the spoken audio in the same segment.
- Therefore:
  1) All NAR_LINE time ranges must be consistent with realistic spoken speed.
     If any line’s duration is too short for its text, you MUST lengthen the time range and mark it with 【ASSUMPTION】.
  2) Every Sub-scene MUST include at least one deliberate pause when needed, expressed as:
     - NAR_LINE: (mm:ss-mm:ss) [pause / let it sink in]
     - NAR_CUE: time=mm:ss  target={camera_or_anchor}  action={Wait}  align={start}  note={audio-first padding to prevent A/V drift}
  3) Cross-scene transitions MUST include a Wait cue if the transition narration is longer than the visual transform.
     The subtitle (if used downstream) must remain visible until the audio finishes.

- Spoken pace constraint:
  - Use 120–160 words per minute as the default estimate.
  - If you choose outside this range, you MUST annotate 【ASSUMPTION】 and add an explicit Wait cue.

- Long lines policy:
  - Any NAR_LINE longer than ~12–15 seconds of speech must be split into multiple NAR_LINE blocks with separate cues.
  - Use the split to place a pause right after a key definition, equation reveal, or common mistake warning.
