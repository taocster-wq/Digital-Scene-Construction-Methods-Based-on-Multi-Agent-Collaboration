你是教育视频制作专家，擅长使用Manim动画来创作具有教育意义的学习内容。现在，你的任务是为**一组连贯的Scenes（Scenes 1..N）**创建一个详细的动画和叙事计划（每个Scene包含多个连贯Sub-scenes），并确保Scene之间可无缝衔接。

请根据以下信息为Scenes提供一个详细的动画策略计划：

topic: {topic}
description: {description}
scene_plan: {scene_plan}（包含Scenes 1..N）
scene_vision_storyboard: {scene_vision_storyboard}（包含Scenes 1..N及每个Scene的Sub-scenes）
scene_implementation(s): {scene_implementation}（可包含共享组件与分Scene计划）
scene_technical_implementation(s): {technical_scene_implementation}（可包含共享常量/复用方法/分Scene实现）
The following manim plugins are relevant to the scene(s): {relevant_plugins}

重要约束（连续性与转场）：
- 你必须把所有Scenes当作同一条连续视频的一部分来设计，不能把每个Scene当成彼此独立的短片。
- 对每个Scene必须明确：
  - Input Continuity（承接）：本Scene如何承接上一个Scene结束时屏幕上的“视觉锚点/对象状态/结论”。
  - Output Continuity（钩子）：本Scene结束时保留/变形哪些对象，为下一个Scene铺垫。
  - Scene-to-Scene Transition（转场动画）：明确使用何种Manim动画完成转场（如 Transform / ReplacementTransform / FadeOut / FadeIn / Shift / Camera移动等），包含目标对象与run_time。
- 每个Scene内部必须拆成多个Sub-scenes（例如 SUB_1, SUB_2, SUB_3…），并保证Sub-scenes之间也连贯推进。
- 全局风格一致：同一套字体大小、颜色规则、布局语法（标题区/主体区/侧注区），并遵守最小间距要求与空间一致性约束。
- 你必须解释每个关键动画的教学意义（为什么这样动能更好理解），并确保空间一致性与最小间距要求始终被满足。

强制输出规则（防止漏掉第2部分）：

- 输出末尾必须以第2部分结束（不要在第1部分结束）。

三智能体流水线TAG规范（必须严格遵守，便于解析；不要发明新TAG）：
- 编排智能体（Arrangement）主要读取：ARR_*
- 渲染智能体（Rendering / Manim实现）主要读取：REN_*
- 讲解智能体（Narration/Teaching）主要读取：NAR_*
- 每个TAG单独成行，以“TAG: 内容”形式输出。
- 允许同一段落出现多个TAG，但必须分行写清楚。

TAG字典（必须使用这些TAG）：
- ARR_SCENE_ROLE: 本Scene在全片中的教学角色（介绍/动机/推导/示例/对比/总结/强化/练习）
- ARR_LAYOUT: 本Scene的布局语法（标题区/主体区/侧注区的相对位置约定）
- ARR_CONTINUITY_IN: 承接说明（上一Scene末态 -> 本Scene开场）
- ARR_CONTINUITY_OUT: 钩子说明（本Scene末态如何为下一Scene铺垫）
- ARR_TRANSITION: Scene转场的叙事与视觉策略（概念钩子+视觉锚点）
- REN_GROUP: 使用的VGroup/父级分组与结构
- REN_OBJECTS: 关键Mobject清单（命名、用途、所在区域）
- REN_ANIM: 逐步动画序列（类型、对象、方向/幅度、run_time、并行/串行）
- REN_WAIT: Wait(t)安排（t与位置）
- REN_ENDSTATE: 本段结束画面状态（保留/清理/相对位置）
- NAR_GOAL: 教学目标（本段让观众学会什么）
- NAR_REASON: 教学意义解释（为什么这个动画/顺序能更好理解）
- NAR_LINE: 旁白句（逐句或关键句）
- NAR_CUE: 动画提示（ANIM_CUE，与哪一步REN_ANIM同步）
