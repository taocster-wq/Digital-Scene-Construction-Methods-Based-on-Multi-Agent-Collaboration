========================

任务：
通读整个技术实现计划（TIP），识别其中描述或引用到的所有 Manim 代码元素，并按指定 JSON 格式输出。
同时使用 <MANIM_REFERENCE> 作为“校验与受限补全”参考，但不得引入 TIP 未触发的新元素。

输出格式（必须严格保持，不得新增字段）：
{
  "classes_and_methods": [
    {"class": "...", "methods": ["__init__", "..."]},
    {"function": "..."},
    {"constant": "..."}
  ]
}

========================
总原则（硬约束）
========================
- 仅从 TIP 抽取；不得基于常识/经验或参考资料补充 TIP 未触发的新条目（Scene 隐式必备与白名单语义函数除外）。
- 不提取占位符与模板字段名（如 {topic}, {description}, {technical_scene_implementation} 等）。
- 不提取 XML/HTML 标签名。
- 不提取纯叙述性文字（除非包含可识别的类/方法/函数/常量）。
- 最终输出必须只包含 JSON，不得包含任何说明/注释/Markdown。

========================
0) 使用参考资料的方式（必须遵守）
========================
0.1 标准化与纠错（允许）
- 若 TIP 中出现近似名称（大小写/拼写轻微错误），且参考资料中存在唯一合理匹配，则输出标准名称。
- 若无法唯一匹配（存在多种可能），则保持 TIP 原样且仅在其看起来像标准类名/方法名/常量名时提取；否则忽略。

0.2 归属与分类校验（允许）
- 若 TIP 提到 class + method，但 method 是否属于该 class 不确定，可用参考资料校验：
  - 若支持归属，保留；
  - 若明确不支持，且 TIP 也无其他依据，则不要硬补（只保留 class 的 __init__ 或默认补全）。
- 参考资料可用于判断某名称更像 Animation/Mobject/Scene/Camera，以避免把动画当作对象类、或把 Wait 当作类等。

0.3 受限补全（允许且受限）
- 参考资料只能帮助补全“TIP 已触发的 class 的 methods”，且必须仍遵守下文 1.2 的默认补全规则范围。
- 参考资料不能产生新的 function/constant/class（除下文允许的 Scene 隐式必备与白名单语义函数）。

========================
1) 类（Class）抽取规则
========================
1.1 识别类名
- 抽取 TIP 中明确出现的 Manim 类名（例如 Scene, Tex, MathTex, Text, VGroup, Group, Line, Dot, Circle, Square, Rectangle, Axes, NumberPlane, SurroundingRectangle 等）。
- 抽取动画类（例如 Create, Write, FadeIn, FadeOut, Transform, ReplacementTransform, GrowFromCenter 等）。
- 隐式必备类：Scene（即使 TIP 未出现，也允许加入）。

1.2 方法抽取与默认补全
- 每个 class 的 methods 必须包含 "__init__"。
- 若 TIP 中出现/暗示使用某方法，则加入 methods（例如 next_to / move_to / shift / to_edge / to_corner / align_to / arrange / scale / rotate / set_color / set_fill / set_stroke / set_opacity / set_width / set_height 等）。
- 参考资料可用于校验方法名、修正大小写、确认方法存在；但不得超出默认补全规则的边界。
- 默认补全规则（只对以下类生效；其他类不做无依据补全）：
  - Scene -> ["__init__", "construct", "play", "wait"]
  - Tex / MathTex -> 默认包含 ["__init__", "next_to", "align_to"]
  - VGroup -> 默认包含 ["__init__"]
  - Shapes（Square/Rectangle/Circle/Polygon 等）-> 默认包含 ["__init__", "set_fill", "set_stroke"]
    - Line 至少包含 ["__init__", "set_stroke"]
  - Animations（FadeIn/FadeOut/Transform/Create/Write/GrowFromCenter 等）-> 默认包含 ["__init__"]

1.3 禁止错误归类
- 不要把方法当作类：Wait 不是类；Wait()/等待语义应映射为 Scene.wait（因此不要输出 {"class":"Wait",...}）。
- 不要把纯概念词当作类（如 “layout zone” “safe area” 等）。

========================
2) 函数（Function）抽取规则
========================
2.1 明确函数名
- 提取 TIP 中明确出现的自定义函数/辅助函数/构造器/校验函数名称：
  - 例如出现 def xxx / xxx(...) / “调用 xxx 函数” 等，即提取 xxx 作为 function。

2.2 语义触发白名单函数（严格白名单）
- 若 TIP 中出现“可复用/模块化/辅助方法/构造器/检查函数/布局分区/转场构建”等语义但未给出函数名，
  允许从下列白名单中选择最匹配者加入输出。
- 只允许使用白名单内名称；严禁输出白名单外函数名。

标准函数白名单（仅语义触发时可添加）：
- create_title
- create_side_note
- create_theorem_block
- create_axes
- create_callout
- group_layout_zones
- enforce_safe_area
- enforce_min_spacing
- prevent_text_overflow
- validate_bounding_boxes
- build_scene_objects
- animate_scene_flow
- build_scene_transition

========================
3) 常量（Constant）抽取规则（颜色只输出名字，不输出hex）
========================
3.1 方向常量
- ORIGIN, UP, DOWN, LEFT, RIGHT, UL, UR, DL, DR
- 仅当 TIP 文本出现或明确暗示使用（如“向上偏移/放到左上角/移动到原点”等）时提取对应常量。
- 禁止输出方向向量数值（如 [0,1,0] 这种）。

3.2 颜色常量（强规则：只输出颜色名字，不输出 hex）
- 若 TIP 文本出现颜色名字常量（如 BLACK, WHITE, RED, GREEN, BLUE, YELLOW 或 BLUE_A/BLUE_B/... 等），直接输出该颜色名字。
- 若 TIP 文本出现 hex 色值（形如 #RRGGBB）：
  - 禁止输出任何以 "#" 开头的字符串；
  - 必须将该 hex 映射为最接近的 Manim 颜色名字后输出（RGB 欧氏距离最近邻；若无法可靠映射则忽略该颜色）。

3.3 数值约束与字体数值（优先编码为语义常量）
- 当 TIP 文本出现明确的关键约束语义时，使用下列语义常量输出（只添加命中的）：

A) 空间约束
- safe area 0.5 -> "SAFE_AREA_MARGIN_0_5"
- min spacing 0.3 -> "MIN_SPACING_0_3"
- 提到 safe area/边距检查 -> "CHECK_SAFE_AREA"
- 提到 spacing/避免重叠 -> "CHECK_MIN_SPACING"

B) 文本溢出策略
- 提到换行/多行 -> "TEXT_WRAP"
- 提到缩放字体 -> "TEXT_SCALE_DOWN"
- 提到侧注区/分栏 -> "TEXT_IN_SIDE_ZONE"
- 提到 bbox/边界框检查 -> "CHECK_TEXT_BBOX"
- 提到 Tex/MathTex 分工 -> "TEX_MATHTEX_POLICY"

C) 分区布局（若 TIP 提到分区）
- 标题区 -> "ZONE_TITLE"
- 主体区 -> "ZONE_MAIN"
- 侧注区 -> "ZONE_SIDE"

D) 连续性/转场
- 提到 anchor/锚点对象 -> "HAS_ANCHOR_OBJECT"
- 提到 Transform 链/对象演化 -> "ANCHOR_TRANSFORM_CHAIN"
- 提到 Scene 转场 -> "HAS_SCENE_TRANSITION"
- 提到 Wait/停顿缓冲 -> "HAS_WAIT_BUFFERS"

- 除上述语义常量外，不要为了“看起来完整”而输出大量裸数字常量；只有当 TIP 明确将某数字当作常量并命名时才输出。

========================
排除规则（必须执行）
========================
- 不输出占位符与模板字段名。
- 不输出 XML/HTML 标签名。
- 不输出任何 hex 色值（"#......"）。
- 不输出白名单外的“语义推断函数名”。
- 不把方法当作类（尤其：Wait 不是类）。

========================
去重与合并（必须执行）
